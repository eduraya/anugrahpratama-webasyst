Компонент общего назначения .tablebox является реализацией табличной верстки через CSS, который актуален до сих пор. Его неоспоримым преимуществом является возможность жестко привязать ширину колонки вне зависимости от контента внутри.

В отличие от .flexbox, контент никогда не сможет повлиять на ширину общего блока и никогда его не растянет, если контент не помещается. Данное качество позволяет делать «неломающуюся» разметку общих глобальных частей сайта, не опасаясь, что часть контента может не поместиться внутри.

Есть и минус — у колонок display: table-cell; не работает свойство position:relative|absolute;. Имейте это ввиду при проектировании скелета. Одно из решений в этой ситуации — использование дополнительной внешней/внутренней обертки.

Никакого специфического для бекенда Webasyst оформления этот класс не навязывает, но будет полезен в качестве структурного контейнера, содержащего внутри разные интерфейсные компоненты: заголовки, меню, кнопки, текстовые поля, блок с произвольным контентом .box.

.tablebox.space-16

С помощью дополнительных классов .tablebox.space-XX добавляются внутренние отступы для дочерних элементов, где XX — значение в пикселях с шагом 4: 0, 4, 8, 12, 16, 20, 24, 28, 32, 36, 40, 44, 48. Шаг в 4 пикселя выбран исходя из соображений сохранения визуальной сетки интерфейса и практической полезности. Пиксели будут пересчитаны в rem.

{_render class="space-16"}
Показать код 
{_render|escape class="space-16"}
